workbox-routing.Route

class   static

A Route consists or a matcher and a handler. A matcher needs to determine if a route should be used for a request. A handler returns a response to the request if there's a match.

handlerCallback

handlerCallback(context) returns Promise containing Response

The handler callback is called when a new service worker fetch event has been matched by a Route. This callback should return a Promise that resolves with a Response.

If a value is returned by the match callback it will be passed in as the params argument of this callback.

This callback is passed into the Route Constructor.

matchCallback

inner

matchCallback(context) returns (Object or null)

The match callback is used to determine if a new service worker fetch event should be handled this Route. Returning a truthy value means this Route will handle and respond to this fetch event.

Return null if this Route shouldn't match against the fetch event.

Any truthy value returned by this callback will be passed to the Route's handler callback.

This callback is passed into the Route Constructor.

Parameter

context

Object
Returns

(Object or null) To signify a match, return anything other than null. Return null if the route shouldn't match. If you return an Object with contents, it will be passed to the Route's handler.
