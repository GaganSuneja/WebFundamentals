workbox-routing.Router

class   static

The Router takes one or more Routes and passes each fetch event through it's routing logic to determine the appropriate way to respond with a Request.

If no route matches a given a request, the Router will use a "default" handler if one is defined.

Should any of the Route's throw an error, you can define a "catch" handler to gracefully deal with these issues and respond with a Request.

If a request matches multiple routes, precedence will be given to the earliest registered route.

If a Route throws an error while handling a request, this handler will be called and given a chance to provide a response.

Parameter

(function() or module:workbox-runtime-caching.Handler)

This parameter can be either a function or an object with a handle function. See the Handler interface for details.

setDefaultHandler

setDefaultHandler(handler)

An optional handler that's called when no routes explicitly match the incoming request.

If the default is not provided, unmatched requests will go against the network as if there were no service worker present.

Parameter

handler

(function() or module:workbox-runtime-caching.Handler)

This parameter can be either a function or an object with a handle function. See the Handler interface for details.

Either option should result in a Response object, which will be used to respond to the fetch event.
